# Архитектура IMock

IMock состоит из backend API, frontend-приложения и локальной базы данных SQLite.

## Backend

Backend реализован на FastAPI и async SQLAlchemy.

Ключевые слои:

- `app/api/endpoints` - HTTP endpoints;
- `app/models` - SQLAlchemy-модели;
- `app/schemas` - Pydantic-схемы API;
- `app/services` - бизнес-логика интервью, LLM и генерации вопросов;
- `app/db` - подключение к БД и локальная подготовка схемы.

## Основные сущности

- `User` - пользователь или администратор.
- `InterviewType` - тип собеседования, например "Backend Java-разработчик".
- `Question` - вопрос банка с эталонным ответом и критериями оценки.
- `QuestionSource` - источник вопроса.
- `QuestionGenerationJob` - задание генерации банка вопросов.
- `InterviewSession` - сессия прохождения интервью.
- `Message` - сообщения кандидата, AI-интервьюера и системы.
- `InterviewResult` - итоговая оценка, сильные стороны, зоны роста и рекомендации.
- `UsageLimit` - техническая защита от неконтролируемого расхода LLM API.

## Интервью

1. Пользователь запрашивает активные типы собеседований.
2. Пользователь выбирает тип и уровень.
3. Backend проверяет наличие активных вопросов в банке.
4. Backend создаёт `InterviewSession`.
5. Первый вопрос берётся из таблицы `questions`.
6. Ответ пользователя сохраняется с привязкой к `question_id`.
7. LLM получает вопрос, эталонный ответ, критерии и историю.
8. Если ответ неполный, AI задаёт уточнение.
9. После завершения создаётся `InterviewResult`.

## Генерация вопросов

1. Администратор запускает генерацию.
2. Backend создаёт `QuestionGenerationJob`.
3. LLM получает prompt с требованием вернуть JSON.
4. Backend валидирует JSON.
5. Вопросы сохраняются в `questions`.
6. Источники сохраняются в `question_sources`.
7. Job получает статус `completed` или `failed`.

## Frontend

Frontend реализован на React, TypeScript, Vite и Material UI.

Основные страницы:

- `/dashboard` - пользовательская панель и запуск интервью;
- `/interviews/:id` - чат с AI-интервьюером;
- `/interviews/:id/result` - результат интервью;
- `/profile` - профиль и подготовка;
- `/admin/interview-types` - управление типами собеседований;
- `/admin/questions` - банк вопросов;
- `/admin/question-generation-jobs` - статус генерации.

## Удалённый legacy

Старая параллельная ветка свободного диалога удалена. Основной и единственный сценарий проведения mock-собеседований работает через `/interviews`.

## Коммерческая логика

В проекте нет тарифов, подписок, оплат и платных планов. Технические лимиты используются только для защиты AI-интеграции.
