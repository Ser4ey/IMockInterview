# Архитектура приложения

Приложение состоит из двух слабосвязных микросервисов *Backend* и *Frontend* взаимодействующих между собой по RestAPI.

## Backend
Стек: Python + FastAPI

*Backend* отвечает за всю логику приложения, когда *Frontend* необходимо сделать что либо,
он отправляет запрос на соответвующий эндпоинт в *Backend*.

### Список эндпоинтов для Rest:
1) Эндпоинты пользователя
- авторизация пользователя (авторизует пользователя и отправляет ему ссеонный токен)
- создание нового аккунта
- информация о пользователе: сколько пользователь сделал запросов, какой у него тарифный план
- повышение тарифного плана пользователя

2) Эндпоинты собеседований (чатов)
- Список всех активных чатов пользователя
- Создание чата с определённой темой собеседования (процесс создания собеседования)
- Информация о чате (его статус)
- Список всех сообщений в текущем чате
- Список n последних сообщений в текущем чате
- Отправка нового сообщения в чат (в запросе передаётся сообщение пользователя, в ответ на запрос приход ответ полученный от ИИ)
- Информация о статистеке пользователя

### Взаимподействие с ИИ
Тебе необходимо написать высокоуровневый класс для работы с ИИ, таким образом, чтоб можно было легко менять конкретный ИИ.
По умолчанию в проекте будет использоваться ИИ от https://yandex.cloud/ru

Такде необходимо написать системные промпты для следующих действий ИИ. (для каждого действия должен быть отдельный метод в высокоуровневым классом для работы с ИИ).
Что долден уметь делать ИИ:
- Сбор информации (вопросов) для проводения собеседования по теме (благодаря MCP). Метод получает название лолжности и уровень кондидата и должен вернуть список вопросов по этой теме.
- Собеседование (все этапв собеседования должны проводится в 1 контексте диалога). Этап знакомства.
- Собеседование (все этапв собеседования должны проводится в 1 контексте диалога). Проверка хард скилов
- Собеседование (все этапв собеседования должны проводится в 1 контексте диалога). Проверка софт скилов
- Собеседование (все этапв собеседования должны проводится в 1 контексте диалога). Результат (модель провидит анализ собеседования и говорит пользователю его сильные и слабые стороны).



## Frontend
Стек: TypeScript + React

Сайт должен иметь следующий функционал:
1) Главная страница 
	- Общая информация о проекте
	- Кнопти входа и регистрации
2) Страницы входа/регистрации
3) Страница с собеседованием
	- Отображение всех собеседований
	- Создать новое собеседованием
4) Страница конкретного собеседования (чата)
5) Личный кабинет пользователя с прогерссом
